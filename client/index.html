GUIA DE CONFIGURACION

Conecta el tenant de tu cliente en 5 pasos

BeAudit usa una App Registration de Azure con permisos de solo lectura. Sin acceso a correo, archivos ni datos personales. Conforme RGPD. El cliente ejecuta un script de PowerShell que prepara el acceso en menos de 2 minutos.

1
El cliente descarga el script de PowerShell
Proporciona al cliente el fichero BeAudit-Setup.ps1. Lo puede ejecutar desde cualquier PC con Windows y PowerShell 5.1+. Solo necesita una cuenta con rol Global Administrator en el tenant.
2
Ejecuta el script (2 minutos)
El script crea automaticamente una App Registration llamada BeAudit Diagnostico, asigna los permisos de lectura necesarios y genera el secreto de acceso. Ningun permiso de escritura — todo es de solo lectura.
PowerShell.exe -ExecutionPolicy Bypass -File BeAudit-Setup.ps1
3
El script muestra 3 valores
Al terminar, el script imprime el Tenant ID, el Client ID (Application ID) y el Client Secret. El cliente los copia y te los envía por email o mensaje seguro.
Permisos que se solicitan (solo lectura):
Organization.Read.AllUser.Read.All Directory.Read.AllPolicy.Read.All UserAuthenticationMethod.Read.AllSecurityEvents.Read.All DeviceManagementManagedDevices.Read.AllSites.Read.All
4
BeServices configura el .env del servidor
Con los 3 valores recibidos, actualizas las variables TENANT_ID, CLIENT_ID y CLIENT_SECRET en el fichero .env del servidor BeAudit. Las credenciales nunca viajan al navegador — solo el servidor las usa.
5
Lanzas el diagnostico en la reunion
Abre BeAudit en el navegador con el cliente delante y pulsa Iniciar diagnostico. En 20-30 segundos tienes el dashboard completo con datos reales del tenant.
Alternativa manual
El cliente puede crear la App Registration manualmente desde portal.azure.com → Microsoft Entra ID → Registros de aplicaciones
